01

项目概述

项目背景

英语学习市场中，背单词仍是用户最高频的刚需场景。大量学习者在阅读英文文章、做题、看剧时遇到生词，传统做法是手动输入查词→添加到生词本→导出到背词软件，流程繁琐且效率低下。OCR拍照识词技术日趋成熟，但市场上缺乏一款"拍照即识、一键导出到主流背词软件"的轻量工具。

产品定位

"词库工厂" — 不做自建背词功能，聚焦"拍照识词→整理→导出"的核心链路，成为各类背词软件的上游"词库供应商"。

核心价值主张

拍照即识 · 三秒成库 · 一键导出到你的专属背词App

主流App导入兼容性分析

调研背景

为确保"词库工厂"定位真正落地，对市场上 7 大主流背词软件逐一调研其单词导入/导出机制，确保本产品的导出格式与各App的导入方式完全匹配。以下表格梳理了各App的导入方式、文件格式要求及操作路径。

各App导入方式详表

背词AppDAU(估)支持的导入方式文件格式要求导入路径注意事项
墨墨背单词 ~800万 批量文件导入、网页端手动输入 TXT（每行一个单词）、CSV、Excel App内「选词→我的词库→+→加入词库」 直接选择文件即可，最便捷的导入体验
不背单词 ~500万 网页端上传自定义词书 TXT（每行一个单词，UTF-8编码） 电脑登录不背官网 →「自定义词书→上传文件」 每个账号限10本DIY词书，通过网页端操作
欧路词典 ~300万 文件导入、网页端导入 XML (StudyList.xml)、TXT、CSV App内「工具→导入生词本」，或 my.eudic.net 支持四种格式导入，最灵活
Anki ~600万 文件导入 CSV/TXT（逗号/分号/制表符分隔）、APKG牌组包 「文件→导入」，设置分隔符匹配 需手动设置字段映射和分隔符
百词斩 ~1500万 创建自定义词书（复制粘贴） TXT（每行一个单词，或逗号分隔） 「单词本→新建单词本」→复制粘贴内容 不支持直接上传文件，需手动复制
有道词典 ~1亿 文件导入 XML格式生词本、电脑端还支持doc/txt 「单词本→设置→导入/导出→导入单词」 仅XML格式导入最稳定
Quizlet ~300万(全球6000万) 网页端导入 CSV两列（term, definition） 「创建学习集→导入」→粘贴或上传CSV 国际用户广泛，支持CSV一键导入

附录C：公开数据库与数据源

本项目整合 5 大公开数据库/开源资源，构建完整的词典查询、语义分析和词频统计能力。所有数据源均为开源或学术用途免费，确保合规性和可持续性。

数据源总览

序号数据源类型规模许可证用途
1ECDICT英汉词典数据库~150万词条，18字段MIT单词释义、音标、词性、考试标签、词频
2NLTK WordNet英语语义词典~155,000单词，~117,000同义词集WordNet 3.0 (免费商用)同义词、反义词、上位词、语义关系
3spaCy en_core_web_sm统计NLP模型300维GloVe向量MIT词性标注、依存分析、词形还原
4PaddleOCR PP-OCRv3OCR识别引擎80+语种Apache 2.0图片英文文本识别
5COCA / BNC英语语料库10亿+词次 (COCA)学术引用词频统计（通过ECDICT间接使用）

1. ECDICT — 英汉词典数据库

来源：skywind3000/ECDICT | 许可证：MIT | 词条数：~150万（完整版）/ ~76万（CSV基础版）

ECDICT 提供丰富的单词释义、词频统计和考试大纲标注，采用 SQLite / CSV / StarDict 多格式分发，适合嵌入应用离线使用。核心字段如下：

字段类型说明示例值
wordVARCHAR(64)单词名称（主键，不区分大小写）"algorithm"
phoneticVARCHAR(64)国际音标"/ˈælɡərɪðəm/"
definitionTEXT英文释义"a set of rules for solving..."
translationTEXT中文释义（分号分隔多义项）"算法；计算程序"
posVARCHAR(16)词性"n"
collinsINTEGER柯林斯星级（0-5星）3
oxfordINTEGER牛津3000核心词汇标识0 或 1
tagTEXT考试标签（空格分隔）"cet6 ky"
bncINTEGERBNC词频顺序数字越小越常用
frqINTEGERCOCA词频顺序数字越小越常用

考试标签覆盖

考试标签词汇量（约）考试标签词汇量（约）
中考zk~1,600考研ky~5,500
高考gk~3,500雅思ielts~6,000
四级cet4~4,500托福toefl~6,500
六级cet6~5,500GREgre~8,000

2. NLTK WordNet — 英语语义词典

来源：Princeton University | 许可证：WordNet 3.0 License（免费用于科研和商业） | 规模：~155,000 单词，~117,000 同义词集，~207,000 词义对

语义关系类型

关系说明示例
同义关系同一synset内{car, auto, automobile}
反义关系意义相反hot ↔ cold
上下位关系IS-A 层级dog → canine → animal
部分整体PART-OFwheel → car

本项目使用的WordNet功能

  • get_synonyms(word) — 同义词列表（最多20个）
  • get_antonyms(word) — 反义词列表（最多10个）
  • get_hypernyms(word) — 上位词链
  • get_word_semantic_info(word) — 完整语义信息（释义+例句+POS）

3. spaCy en_core_web_sm — 统计NLP模型

来源：Explosion AI | 许可证：MIT | 训练数据：OntoNotes 5.0, CoNLL-2003 | 词向量：300维 GloVe (Common Crawl) | 架构：多任务CNN

流水线组件：tagger（词性标注）, parser（依存句法分析）, ner（命名实体识别）, attribute_ruler, lemmatizer（词形还原）。

4. PaddleOCR — OCR文字识别引擎

来源：PaddlePaddle/PaddleOCR | 许可证：Apache 2.0 | 模型版本：PP-OCRv3（英文识别模型）

属性详情
英文识别提升相比 PP-OCRv2 提升 11%
支持语种80+ 种语言
检测算法DB (Differentiable Binarization)
识别算法SVTR (Scene Text Recognition), IJCAI 2022
最新版本PaddleOCR 3.2.0 (2025.08) / PP-OCRv5

7. 数据初始化流程

# 全自动下载并安装所有数据
cd backend
python setup_data.py

# 跳过指定数据源
python setup_data.py --skip-nltk
python setup_data.py --skip-spacy

# 仅验证/生成报告
python setup_data.py --validate
python setup_data.py --report

运行后将自动：下载ECDICT SQLite（~150万词条）、下载NLTK数据包（WordNet/punkt/词性标注器）、下载spaCy英语模型（en_core_web_sm）、生成 data_report.json 数据总览报告。
